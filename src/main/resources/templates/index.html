문제

  1. 팀장은 프로젝트를 다운로드받아 main브랜치를 생성한 다음 push
  2. 팀원은 clone한 다음 feature/이름 브랜치를 만든 다음 작업
  3. css 폴더에 있는 contact, supply, todo 폴더에 각자 자신의 css를 추가해서 적용
  4. header, nav, footer, aside 파일은 fragment의 것을 사용
  5. contact, supply, todo 패키지에 있는 Contact, Supply, Todo 클래스를 참고해 작업할 테이블을 생성
  6. 자신의 작업이 끝나면 main브랜치에서 업데이트한 다음 push
  7. main에 모두의 작업이 합쳐지면 db를 원격으로 변경한 다음 render에 배포
  8. Dockerfile은 포함되어 있음
